¬øQU√â ES localStorage?
--------------------------------------------------------------------------------
- Almacenamiento local del navegador web
- Los datos persisten al cerrar el navegador
- Solo accesible desde el mismo navegador/computadora
- Capacidad: ~5-10 MB
- Solo almacena strings (por eso usamos JSON)

UPDATE (Actualizar):

  p.id === id ? { ...p, entregado: true } : p

¬øPOR QU√â JSON.parse y JSON.stringify?

localStorage solo guarda strings, entonces:

JSON.parse: String JSON ‚Üí Objeto JavaScript

üîë CONCEPTOS CLAVE DE REACT
================================================================================

1. COMPONENTES
--------------------------------------------------------------------------------
- Bloques reutilizables de c√≥digo
- Pueden ser funciones (componentes funcionales)
- Reciben props y retornan JSX
- Ejemplo: Navbar, Formulario, Catalogo

2. JSX (JavaScript XML)
--------------------------------------------------------------------------------
- Sintaxis que parece HTML pero es JavaScript
- Se compila a React.createElement()
- Permite escribir interfaz de usuario de forma declarativa
- Ejemplo: <div className="container">Hola</div>

3. HOOKS (desde React 16.8)
--------------------------------------------------------------------------------
Funciones especiales que permiten usar estado y otras caracter√≠sticas de React:

useState:
- Agrega estado local al componente
- Retorna [valor, funcionParaActualizarlo]
- Ejemplo: const [pedidos, setPedidos] = useState([]);

useEffect:
- Ejecuta efectos secundarios (side effects)
- Se ejecuta despu√©s del render
- Puede tener dependencias que controlan cu√°ndo se ejecuta
- Puede retornar funci√≥n de cleanup
- Ejemplo: cargar datos, suscripciones, timers

useForm (de react-hook-form):
- Hook personalizado (custom hook)
- Simplifica manejo de formularios
- Proporciona register, handleSubmit, errors, reset

4. PROPS
--------------------------------------------------------------------------------
- Propiedades que se pasan de componente padre a hijo
- Son inmutables (read-only)
- Ejemplo: <Navbar logo={logoUrl} />

5. ESTADO (State)
--------------------------------------------------------------------------------
- Datos que cambian en el tiempo
- Cuando el estado cambia, React re-renderiza el componente
- Ejemplo: pedidos, nombreBusqueda, tipoProducto

6. EVENTOS
--------------------------------------------------------------------------------
- onClick, onChange, onSubmit, etc.
- Usan camelCase (no onclick)
- Ejemplo: <button onClick={handleClick}>

M√âTODOS PRINCIPALES:
